# 一、标准版宣传文案（适合知乎  B 站长文）

UMBL（Universal Meaning‑Based Language）是一个正在开发中的“象形语言设计项目”。  
它不是图标库，也不是 UI 设计，而是一种尝试从零开始构建“象形文字语言”的工程化实验。

为什么要做这件事？  
灵感来自对中文和英文长期的对比观察：

- 中文有造字传统、偏旁部首、语义透明度  
- 英文依赖词根、词缀构造抽象概念  
- 中文媒体文化天然偏向视觉表达（字幕、弹幕）  
- 表音语言在跨文化理解中存在局限  
- 自然语言在抽象表达和逻辑一致性上并不稳定  

这些现象让我产生一个问题：  
如果我们重新设计一种语言，它能否既像中文一样“看得懂”，又像工程系统一样“可组合、可扩展”？

UMBL 的目标不是替代任何语言，也不是证明自己更好。  
它只是一个开放的、可实验的语言原型，用来探索：

- 如何构建象形语素  
- 如何组合成象形词  
- 如何形成象形句  
- 如何设计语法、构字规则和演进机制  

目前 UMBL 仍处于早期阶段，但它已经能作为：

- 本科论文选题  
- 语言学  认知科学  设计  计算机科学的研究对象  
- 一个关于“语言还能怎样被设计”的讨论起点  

如果你对语言、符号、视觉表达、跨文化沟通或构造语言感兴趣，欢迎关注这个项目。

# 二、轻量版宣传文案（适合小红书  B 站简介）

UMBL 是一个正在开发中的“象形语言设计项目”。  
它不是图标库，而是尝试从零开始构建一种“可读、可组合、可演进”的象形文字语言。

灵感来自中英文表达方式的差异：  
中文造字、偏旁、视觉表达；英文词根、构词法；表音语言的局限……  
这些都让我思考：语言能不能被重新设计？

UMBL 目前还不是成熟语言，但它是一个开放的语言原型，适合做研究、做实验、做讨论。  
如果你对语言、符号、视觉表达或构造语言感兴趣，欢迎一起探索。
